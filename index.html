CLI Usage¶

logsweet¶

A suite with a variety of tools for handling log messages.

logsweet [OPTIONS] COMMAND [ARGS]...

Commands

listen

Listen to text lines with ZeroMQ SUB and/or…

mock

Generate random log file entries.

proxy

Run a log proxy between watchers and…

watch

Follow logfiles and send new lines via ZeroMQ…

logsweet watch¶

Follow logfiles and send new lines via ZeroMQ PUB and/or PUSH socket.

logsweet watch [OPTIONS] FILE_GLOB

Options

-b, --bind-address <bind_address>¶

IP and port to bind the ZeroMQ PUB socket.

-c, --connect-address <connect_address>¶

Hostname(s) or IP(s) with port to connect the ZeroMQ PUSH socket.

-cfg, --config-file <config_file>¶

A configuration file (YAML).

-x, --exec-actions¶

Activates the execution of action rules from the configuration.

-a, --all-lines¶

Broadcast all existing content before following.

-t, --tail-lines <tail_lines>¶

Number of tail lines to broadcast before following.

-e, --encoding <encoding>¶

Encoding for reading the text files.

-n, --name <name>¶

The source name for the watched logs.

--echo¶

Print events to the console.

Arguments

FILE_GLOB¶

Required argument

logsweet listen¶

Listen to text lines with ZeroMQ SUB and/or PULL socket.

logsweet listen [OPTIONS]

Options

-b, --bind-address <bind_address>¶

IP and port to bind the ZeroMQ PULL socket.

-c, --connect-address <connect_address>¶

Hostname(s) or IP(s) with port to connect the ZeroMQ SUB socket.

-cfg, --config-file <config_file>¶

A configuration file (YAML).

-x, --exec-actions¶

Activates the execution of action rules from the configuration.

logsweet proxy¶

Run a log proxy between watchers and listeners. Allowing listeners and watchers to come and go. Supporting a high availability architecture with multiple proxies.

logsweet proxy [OPTIONS]

Options

-bb, --backend-bind-address <backend_bind_address>¶

The IP and port to bind a ZeroMQ PULL socket for collecting log messages from watchers or other proxies.

-bc, --backend-connect-address <backend_connect_address>¶

Hostname(s) or IP(s) with port to connect a ZeroMQ SUB socket for receiving log messages from watchers or other proxies.

-fb, --frontend-bind-address <frontend_bind_address>¶

The IP and port to bind the PUB socket for broadcasting log messages to listeners or other proxies.

-fc, --frontend-connect-address <frontend_connect_address>¶

Hostname(s) or IP(s) with port to connect a ZeroMQ PUSH socket for transmitting log messages to a listener or other proxies.

logsweet mock¶

Generate random log file entries.

logsweet mock [OPTIONS] [LOGFILES]...

Options

-i, --interval <interval>¶

The interval for entry generation in seconds.

-n, --max-lines <max_lines>¶

The number of log messages to generate.

Arguments

LOGFILES¶

Optional argument(s)
