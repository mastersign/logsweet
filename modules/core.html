Module logsweet.core¶

This module contains the core functionality.

logsweet.core.listen_and_print(bind_address: Optional[str] = None, connect_addresses: Optional[Sequence[str]] = None, config_file: Optional[TextIO] = None, interval: float = 0.1, exec_actions: bool = False)[source]¶

Connects to watchers and proxies with a ZeroMQ SUB socket and/or binds a ZeroMQ PULL socket for watchers and proxies to connect to; and prints the received lines.

Parameters:
  • bind_address (Optional[str]) – The address to bind the PULL socket to. (Listening for incoming connections from watchers or proxies.)
  • connect_addresses (Optional[Sequence[str]]) – The addresses to connect a SUB socket to. (Actively connecting to watchers or proxies.)
  • config_file (Optional[TextIO]) – The name of a YAML configuration file.
  • interval (float) – The timeout in seconds when waiting for new messages before handling possible interruption.
  • exec_actions (bool) – A switch to activate the execution of actions.

Parameters:

logsweet.core.watch_and_send(file_glob: str, bind_address: Optional[str] = None, connect_addresses: Optional[Sequence[str]] = None, config_file: Optional[TextIO] = None, exec_actions: bool = False, all_lines: bool = False, tail_lines: int = 0, encoding: Optional[str] = None, name: str = 'unknown', echo: bool = False)[source]¶

Start following text files, sending new lines via a ZeroMQ PUB and/or PUSH socket.

Parameters:
  • file_glob (str) – A path to the text file to tail.
  • bind_address (Optional[str]) – The address to bind the PUB socket to; Listening for incoming connections from listeners and proxies.
  • connect_addresses (Optional[Sequence[str]]) – The addresses to connect a PUSH socket to; Actively connecting to one listener or multiple proxies.
  • config_file (Optional[TextIO]) – The name of a YAML configuration file.
  • exec_actions (bool) – A switch to activate the execution of actions.
  • all_lines (bool) – A flag to indicate if already existing content of the file should be broadcasted.
  • tail_lines (int) – Indicates a number of trailing lines to broadcast before following new lines.
  • name (str) – The name of this broadcaster.
  • echo (bool) – If True, new lines are printed to the console.
  • encoding (Optional[str]) – Specifies the encoding for reading the text files. If None defaults to preferred encoding of current user.

  • max_n (Optional[int]) – The maximal number of entries to write.
